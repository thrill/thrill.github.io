22 /*!
23 
24 Command line parser which automatically fills variables and prints nice usage
25 messages.
26 
27 This is a straightforward command line parser in C++, which will recognize short
28 options -s, long options --long and parameters, both required and optional. It
29 will automatically parse integers and <b>byte sizes</b> with SI/IEC suffixes
30 (e.g. 1 GiB). It also works with lists of strings, e.g. multiple filenames.
31 
32 \snippet tests/common/cmdline_parser_example.cpp example
33 
34 When running the program above without arguments, it will print:
35 \verbatim
36 $ ./common_cmdline_parser_example
37 Missing required argument for parameter 'filename'
38 
39 Usage: ./common_cmdline_parser_example [options] <filename>
40 
41 This may some day be a useful program, which solves many serious problems of
42 the real world and achives global peace.
43 
44 Author: Timo Bingmann <tb@panthema.net>
45 
46 Parameters:
47  filename A filename to process
48 Options:
49  -r, --rounds N Run N rounds of the experiment.
50  -s, --size Number of bytes to process.
51 \endverbatim
52 
53 Nice output, notice the line wrapping of the description and formatting of
54 parameters and arguments. These too are wrapped if the description is too long.
55 
56 We now try to give the program some arguments:
57 \verbatim
58 $ ./common_cmdline_parser_example -s 2GiB -r 42 /dev/null
59 Option -s, --size set to 2147483648.
60 Option -r, --rounds N set to 42.
61 Parameter filename set to "/dev/null".
62 Command line parsed okay.
63 Parameters:
64  filename (string) "/dev/null"
65 Options:
66  -r, --rounds N (unsigned integer) 42
67  -s, --size (bytes) 2147483648
68 \endverbatim
69 
70 The output shows pretty much what happens. The command line parser is by default
71 in a verbose mode outputting all arguments and values parsed. The debug summary
72 shows to have values the corresponding variables were set.
73 
74 One feature worth naming is that the parser also supports lists of strings,
75 i.e. \c std::vector<std::string> via \ref CmdlineParser::AddParamStringlist()
76 and similar.
77 
78 */
