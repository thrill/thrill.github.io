30 /*!
31  * A Stream is a virtual set of connections to all other worker instances,
32  * hence a "Stream" bundles them to a logical communication context. We call an
33  * individual connection from a worker to another worker a "Host".
34  *
35  * To use a Stream, one can get a vector of BlockWriter via OpenWriters() of
36  * outbound Stream. The vector is of size of workers in the system.
37  * One can then write items destined to the
38  * corresponding worker. The written items are buffered into a Block and only
39  * sent when the Block is full. To force a send, use BlockWriter::Flush(). When
40  * all items are sent, the BlockWriters **must** be closed using
41  * BlockWriter::Close().
42  *
43  * To read the inbound Connection items, one can get a vector of BlockReader via
44  * OpenReaders(), which can then be used to read items sent by individual
45  * workers.
46  *
47  * Alternatively, one can use OpenReader() to get a BlockReader which delivers
48  * all items from *all* worker in worker order (concatenating all inbound
49  * Connections).
50  *
51  * As soon as all attached streams of the Stream have been Close() the number of
52  * expected streams is reached, the stream is marked as finished and no more
53  * data will arrive.
54  */
