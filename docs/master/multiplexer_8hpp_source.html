50 /*!
51  * Multiplexes virtual Connections on Dispatcher.
52  *
53  * A worker as a TCP conneciton to each other worker to exchange large amounts
54  * of data. Since multiple exchanges can occur at the same time on this single
55  * connection we use multiplexing. The slices are called Blocks and are
56  * indicated by a \ref MultiplexerHeader. Multiple Blocks form a Stream on a
57  * single TCP connection. The multiplexer multiplexes all streams on all
58  * sockets.
59  *
60  * All sockets are polled for headers. As soon as the a header arrives it is
61  * either attached to an existing stream or a new stream instance is
62  * created.
63  */
64 class Multiplexer
